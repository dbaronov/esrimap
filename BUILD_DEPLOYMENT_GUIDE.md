# Build & Deployment Guide

## 🎯 The Issue

When you build and view the project, you might see README files instead of the map application. This is a **configuration issue with how the dev server serves files**.

## ✅ What's Actually Happening

### Build Output
When you run `npm run build`, Webpack correctly creates:
- **`dist/index.html`** - Your app entry point ✅
- **`dist/bundle.js`** - Your React code ✅
- **`dist/*.bundle.js`** - Esri library chunks ✅

### The Fix
The webpack devServer configuration was pointing to the project root instead of the dist folder. This has been fixed.

## 🚀 How to Build & Run

### Development Mode (with hot reload)
```bash
npm start
# or
npm run dev
```
- Opens http://localhost:3000 automatically
- Watches for file changes
- Hot module reloading enabled
- **Serves from:** dist directory

### Production Build
```bash
npm run build
```
- Creates optimized bundle in `dist/`
- Minified code
- Split chunks for efficient loading
- Creates `dist/index.html` entry point

### Serve Production Build Locally
```bash
# Install a simple server
npm install -g serve

# Serve the dist folder
serve -s dist

# Opens http://localhost:3000
```

## 🌐 Deployment Options

### Option 1: Vercel (Recommended)
```bash
npm install -g vercel
vercel --prod
```

### Option 2: Netlify
```bash
# Using CLI
npm install -g netlify-cli
netlify deploy --prod --dir=dist
```

### Option 3: GitHub Pages
```bash
# Install gh-pages
npm install --save-dev gh-pages

# Add to package.json scripts:
"deploy": "npm run build && gh-pages -d dist"

# Then run:
npm run deploy
```

### Option 4: Traditional Server (Nginx/Apache)
```bash
# Copy entire dist folder to your server
scp -r dist/ user@your-server:/var/www/esrimap/
```

**Server Configuration (Nginx example):**
```nginx
server {
    listen 80;
    root /var/www/esrimap;
    index index.html;
    
    # SPA routing - send all requests to index.html
    location / {
        try_files $uri /index.html;
    }
}
```

## ⚠️ Common Issues & Solutions

### Issue: Seeing README files instead of app
**Solution:**
```bash
# Make sure you're serving from the right place
npm run build        # Creates dist/ folder
serve -s dist        # Serves ONLY dist folder
# Open http://localhost:3000
```

### Issue: Blank page or 404 errors
**Solution:**
1. Make sure `index.html` exists: `ls dist/index.html`
2. Make sure `bundle.js` exists: `ls dist/bundle.js`
3. Clear browser cache: Ctrl+Shift+Delete (or Cmd+Shift+Delete)
4. Check browser console for errors (F12)

### Issue: Assets not loading
**Solution:**
1. Check network tab in DevTools (F12)
2. Verify all chunk files are loading
3. Check that Esri CSS is loading from CDN

### Issue: Map not showing
**Solution:**
1. Open browser console (F12)
2. Check for JavaScript errors
3. Verify Esri library is loaded (check Network tab for `arcgis` files)
4. Try `npm start` to run development mode
